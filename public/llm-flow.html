neo-geo · Cómo, cuándo y en qué orden se ejecutan los modelos de lenguaje

1 · ¿Cuándo se disparan?
🖱️

Manual — "Run Now"

El usuario hace clic en el botón de la tabla de prompts. Se crea 1 run por cada LLM habilitado en el workspace.

prompt/run.multi
📦

Importación masiva

Si "Run after import" está activo al importar N prompts, se crean N × M runs (M = LLMs habilitados) al instante.

N × prompt/run.multi
⏰

Cron diario

El job programado runPromptScheduled ejecuta todos los prompts activos de todos los workspaces automáticamente.

Diario · 2am UTC

4 · Control de concurrencia
5

Máximo 5 ejecuciones LLM en paralelo

Inngest gestiona la cola automáticamente. Si hay 50 prompts en espera, los agrupa en batches de 5. Cada run fallido se reintenta hasta 2 veces antes de marcar como "failed".

5 · Pipeline completo por cada run
1

Fetch contexto (paralelo)

Carga en paralelo: prompt.text, workspace, brand propia, competidores, llm_provider, model override de workspace_llm_config.

Promise.all(5 queries) SERVICE_ROLE_KEY
2

Marcar run como "running"

INSERT en prompt_runs o UPDATE status='running' + started_at. El run queda visible en la UI.

status: running
3

Llamar a OpenRouter

POST a openrouter.ai/api/v1/chat/completions con el prompt.text como user message. max_tokens=1000, temperature=0.2 para respuestas consistentes.

OPENROUTER_API_KEY requerida max_tokens: 1000 temperature: 0.2
4

Guardar respuesta cruda

UPDATE prompt_runs con raw_response, model, input_tokens, output_tokens, cost_usd (coste real de OpenRouter), status='completed'.

status: completed cost_usd real
5

detectBrands() — análisis en memoria

Analiza el rawResponse buscando la marca propia y competidores por nombre y alias. Devuelve: posición, sentiment, mention_type, confidence.

primary_recommendation list_option comparison warning confidence score
6

INSERT mentions

Una fila por cada marca detectada (propia + competidores). Incluye: brand_type, position, sentiment, mention_type, confidence.

brand_type: own brand_type: competitor position 1–N
7

extractSourcesFromResponse() + INSERT sources

Extrae URLs mencionadas en la respuesta del LLM. Guarda domain, url, title y marcada como cited_by_llm=true.

cited_by_llm: true domain + URL
8

UPSERT daily_prompt_metrics

Calcula y guarda métricas del día: SOV (share of voice), brand_position, competitor_count, sentiment, consistency_score (% de menciones en los últimos 5 runs).

SOV = propia / (propia + competidores) consistency últimos 5 runs onConflict: upsert
9

UPSERT daily_workspace_metrics

Actualiza las métricas globales del workspace (suma de todos los prompts). Usado en el Dashboard.

métricas globales workspace
10

revalidatePath() — refresca la UI

Invalida la caché de Next.js para /prompts y /dashboard, forzando un re-fetch del server component en el siguiente request.

/[workspace]/prompts /[workspace]/dashboard
6 · Puntos clave
🚫

Sin mocks, sin fallbacks

Si OPENROUTER_API_KEY no está configurada, el código lanza un error explícito. No hay datos simulados, nunca.

💰

Costos reales de OpenRouter

El campo cost_usd se toma directamente de upstream_inference_cost. Solo se estima si OpenRouter no lo devuelve.

🤖

Auto-detección de competidores

Si el LLM menciona una marca no registrada (ej. una aerolínea), se añade automáticamente a competitor_suggestions para revisión.

⚡

Escrituras en paralelo (Fast path)

En executePromptRunFast (para batches), el UPDATE del run + INSERT de mentions + UPSERT de métricas se ejecutan en Promise.all simultáneamente.

🔑

Service Role Key (bypass RLS)

Las funciones Inngest usan SUPABASE_SERVICE_ROLE_KEY para saltarse las políticas RLS. Las Server Actions usan las cookies del usuario.

📊

Consistencia: últimos 5 runs

El consistency_score se calcula sobre los últimos 5 runs completados para ese prompt+LLM. Mide si la marca aparece siempre o de forma irregular.

7 · Cuadro de cuándo se ejecuta cada flujo
🖱️ Usuario · "Run Now" 1 prompt × N LLMs habilitados → N runs en paralelo (máx 5 simultáneos)
📦 Import con run M prompts × N LLMs → M×N runs, disparados en cola Inngest
⏰ Cron diario Todos los prompts activos × todos los workspaces × todos los LLMs habilitados
🔁 Retry automático Máximo 2 reintentos por run fallido. Si sigue fallando: status='failed' + error_message

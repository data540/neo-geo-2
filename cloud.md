# Cloud

## Product scope and client context
- This cloud deployment supports a single client in a single industry vertical: one specific airline.
- The platform must not be treated as multi-tenant across unrelated sectors.
- All prompt orchestration, analytics pipelines, and operational workflows must be airline-specific.

## Market and geography rules
- Primary market: Spain.
- Secondary market: Colombia.
- Prompt logic and analysis should prioritize airline operating realities, customer expectations, and policy context for these two markets.

## Prompt design constraints
- All prompts must assume airline user intent and airline business context.
- Prompt categories should include, at minimum: cancellations, delays, baggage, check-in, boarding, refunds, rebooking, compensation, and disruption handling.
- Avoid generic customer-service prompt templates that could apply equally to other sectors.
- Use airline terminology and decision criteria relevant to flight operations and passenger support.

## Analysis and reporting constraints
- All company analysis must be tied to airline KPIs and flight-support outcomes.
- Route and disruption analysis should emphasize Spain and Colombia network impact.
- Any recommendations generated by prompts or analysis must be actionable for airline operations and support teams.
